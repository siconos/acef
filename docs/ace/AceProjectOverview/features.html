Features

F-1.001: Store a Netlist

Priority: Essential
Functional area(s): Parser
Description: The parser store a Netlist. There are an API to explore the circuit topology.
Notes and Questions:

F-1.002: Sub-circuit management

Priority: Essential
Functional area(s): Parser
Description: Sub circuit is a special key in a Netlist. It consists in managing it when the sub circuit is a comparator.
Notes and Questions: A comparator is a non smooth component modelized with complementarity conditions.

F-1.003: Transient analysis.

Priority: Secondary importance
Functional area(s): Parser
Description: A Netlist has a special key word to define the transient analysis parameters. It consists in reading and exporting these parameters.
Notes and Questions: In current version, this information is written in an other file.

F-2.000: Linear, circuit RLC

Priority: Essential
Functional area(s): ACEF
Description: In this case the circuit is composed only with resistors, capacitors and inductors. It consists in developing an automatic circuit equation formulation. The result is a linear system.
Notes and Questions:


F-2.001: Diodes

Priority: Essential
Functional area(s): ACEF
Description: A diode is a non smooth component, modelized with complementarity conditions. The diode model is an ideal diode. The automatic circuit equation formulation leads to a Mixte Linear Complementarity Problem(MLCP).
Notes and Questions: There is no parameter in the model.


F-2.002: Transistor MOS

Priority: Essential
Functional area(s): ACEF
Description: A transistor is a non smooth component, piecewise linear ,modelized with complementarity conditions. Automatic circuit equation formulation leads to a Mixte Linear Complementarity Problem(MLCP).
Notes and Questions: There is no parameter in the model.


F-2.003: Comparator

Priority: Essential
Functional area(s): ACEF
Description: A Comparator is a non smooth component, piecewise linear ,modelized with complementarity conditions.
Notes and Questions: There are no parameters in the model.


F-2.004: Diode, transistor, comparator with parameters

Priority: Secondary importance
Functional area(s): ACEF
Description: It consists to customize the component model with parameters(Saturation current, zero biais,...). The goal is to be nearer to the physical behaviour.
Notes and Questions: Parameters list must be defiened for each component.


F-2.005: Automatique picewise linear approximation and L.C.P. formulation.

Priority: Essential
Functional area(s): ACEF
Description: The goal is to develop a module to build a picewise linear approximation for any non-smooth component, and write the L.C.P. formulation of this geometry.
Notes and Questions: A feasibility study must be done.


F-2.006: Use simple piecewise linear approximation to accelerate the M.L.C.P. resolution..

Priority: Secondary importance
Functional area(s): ACEF
Description: The dimension of the L.C.P. formulation for a transistor is 10. These 10 parameters described 5 linear parts in the picewise linear description. The goal is to use a very simple picewise linear description, with only two parts, to get a smaller M.L.C.P.. We solve this smaller formulation, and the result is used like a "guess point" to solve the initial M.L.C.P.
Notes and Questions: A feasibility study must be done.


F-2.007: Non linear circuit: R,L or C non constant.

Priority: Essential
Functional area(s): ACEF
Description: It consists in updating the table equations when R, L or C are not constant.
Notes and Questions: A time discretisation for R, L, C must be done.

F-4.002: Write a mixted solver

Priority: Essential
Functional area(s): Mixte Simulator
Description: It consists in writing an iterator algorithm using both Newton-Raphton and MLCP formulation. The non smooth components are described with a LCP formulation.
